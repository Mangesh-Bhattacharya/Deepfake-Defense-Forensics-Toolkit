# Forensic Report Templates

## Standard (template-only) report

Generated by `forensic-reporting/report_generator.py` with no flags. Every field is filled
deterministically from detector output — see `forensic-reporting/templates/forensic_report_template.md`
for the underlying skeleton, and `docs/sample-reports/CASE-0001_forensic_report.md` for a
real generated example.

Sections: header metadata table → evidence description → manipulation markers table →
chain-of-custody reference → methodology reference → disclaimer.

## LLM-assisted variant

Generated with `--llm`. Adds one additional section, **LLM-Assisted Narrative Summary**,
inserted between the markers table and the chain-of-custody section. This section is:

- Always preceded by a blockquote explaining it was LLM-drafted and requires human sign-off.
- Derived only from the same structured markers shown in the table above it (see
  `docs/llm-usage-guide.md` for the enforcement mechanism).
- Never a substitute for the markers table — both are always present together.

Example: `docs/sample-reports/CASE-LLM-DEMO_forensic_report.md`.

## PDF variant

`forensic-reporting/pdf_report.py` renders the same case data as a polished PDF using
reportlab — same sections, same disclaimer, suitable for attaching to an evidence bundle
or emailing to a non-technical stakeholder. Example:
`docs/sample-reports/CASE-DEMO_forensic_report.pdf`.

## Evidence bundle

`forensic-reporting/evidence_packaging.py` wraps the Markdown/JSON/PDF report plus a
`MANIFEST.json` (SHA-256 per file) into a single ZIP — the deliverable unit for handoff to
a case owner or reviewer. `verify_bundle()` re-checks every hash on receipt.

## Chain-of-custody log format

`forensic-reporting/chain_of_custody.py` produces an append-only JSON-lines log where each
entry's hash chains to the previous entry's hash (tamper-evident). Fields: `timestamp`,
`actor`, `action`, `evidence_id`, `evidence_hash`, `notes`, `prev_entry_hash`, `entry_hash`.
`ChainOfCustodyLog.verify_chain()` recomputes every hash and returns `False` the instant
any record has been altered or reordered.
