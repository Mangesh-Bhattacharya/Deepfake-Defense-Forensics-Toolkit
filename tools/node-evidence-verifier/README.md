# Evidence Bundle Verifier (Node.js)

A zero-dependency Node.js companion to `forensic-reporting/evidence_packaging.py`.

Why it exists: a case owner, auditor, or legal reviewer receiving an evidence bundle
shouldn't need Python or this toolkit installed just to confirm the SHA-256 manifest
still matches every file inside the ZIP. An independent verifier written in a different
language than the one that produced the bundle is also a meaningfully stronger integrity
check than re-using the exact code that wrote the file.

Uses only Node.js built-ins (`fs`, `zlib`, `crypto`) — no `npm install` required. It
implements just enough of the PKZIP format (End Of Central Directory, Central Directory,
Local File Header; STORE and DEFLATE methods) to read bundles produced by Python's
`zipfile` module.

## Usage

```bash
node verify_bundle.js ../../docs/sample-reports/CASE-0001_evidence_bundle.zip
```

Exits `0` if every file's hash matches the manifest, `1` otherwise (script-friendly for
CI/automation). On a tampered or corrupted entry it reports `HASH_MISMATCH` or
`CORRUPT_OR_TAMPERED` per file rather than crashing.

## Tests

```bash
npm test
```

Runs against the real bundle in `docs/sample-reports/` (generated by
`forensic-reporting/evidence_packaging.py`) plus a synthetic single-byte-tamper case.
